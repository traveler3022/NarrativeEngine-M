## Conditions for Phase 4

1. **W0 must include smoke test scaffolding** — no logic extraction wave
   may begin until smoke tests exist
2. **Each wave must pass the per-wave gate** (3.3 exit criteria) before
   merge to `main`
3. **Any wave that fails 3 times** in fix-forward mode triggers a
   design re-review (return to Phase 3)
4. **`pre-phase3` tag must remain** until W12 (final validation) is
   complete — it is the nuclear rollback point
5. **Phase 4.1 (Implementation Planning)** must be re-aligned with
   Phase 3.3 wave assignment — the original Phase 4.1 was written
   before Phase 3 existed and may have stale assumptions
6. **Golden Rule (RF-based waves)** — every wave must declare which RF
   cases it closes or advances. A wave without RF cases is invalid and
   must be re-scoped. (Per 3.3)
7. **Traceability Matrix maintenance** — after each wave's merge, the
   matrix in 3.6 must be updated: RF case Status → DONE, Commit →
   merge hash, Result → concrete outcome. A wave is not closed until
   its RF cases show DONE in the matrix. (Per 3.6)
8. **Self-correction audit** — if a wave triggers self-correction
   (RF case turns out not to need its planned solution), the
   Self-Correction Audit Trail in 3.6 must be updated with the
   trigger, action, and new wave assignment. The RF case is either
   re-scoped in 3.1 or moved to a different wave — never silently
   dropped.
