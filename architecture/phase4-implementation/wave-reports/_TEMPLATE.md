# Wave Report Template

**Purpose:** Every wave in Phase 4 ends with a wave report following
this template. Reports live in `architecture/phase4-implementation/wave-reports/`.

## Required Structure

Each wave report MUST start with a **Goal sentence** and end with a
**Result sentence**. These two sentences are the auditable summary;
everything else is supporting evidence.

### Goal sentence (top of report)

```
هدف این Wave: بستن RF-00X تا RF-00Y و حذف V-00A تا V-00B.
```

(For infrastructure waves like W0 that don't close RF cases:
"هدف این Wave: آماده‌سازی زیرساخت برای RF-001 تا RF-007 بدون بستن هیچ RF.")

### Result sentence (bottom of report)

```
نتیجه: X Violation حذف شد، Y RF بسته شد، Z مورد برای Wave بعد باقی ماند.
```

(For W0: "نتیجه: 0 Violation حذف شد، 0 RF بسته شد، 7 RF در وضعیت Prepared باقی ماند.")

## Full Template

```markdown
# Wave W{n} Report — <wave name>

**Merge commit:** <hash>
**Date:** <YYYY-MM-DD>
**Branch:** phase4/w{n}-<name>

## هدف (Goal)

هدف این Wave: <one sentence — RF cases it closes, violations it removes>

## RF Cases in Scope

| RF | Action | Status before | Status after |
|----|--------|---------------|--------------|
| RF-00X | Close | Planned/Prepared | Done |
| RF-00Y | Advance | Planned | Prepared |

## Diff Report

(Generated by `node scripts/wave-diff.mjs W{n} <pre-wave-baseline.json>`)

| Metric | Value |
|--------|-------|
| Baseline before | N violations |
| Baseline after | M violations |
| New | X (should be 0) |
| Resolved | Y |
| Expected | Z (from 3.3 wave assignment) |
| Status | ✅ PASS / ❌ FAIL |

## Behavior Preservation

(For waves that must not change behavior — W0, W1, W2, W3, W7, W8, W9, W10, W11a-d)

| Check | Result |
|-------|--------|
| Smoke tests | N passed |
| tsc -b | PASS/FAIL |
| vite build | PASS/FAIL |
| vitest | N passed (no regressions) |

(For waves that may change internal call patterns — W4, W5, W6, W11e)

| Check | Result |
|-------|--------|
| Pre-extraction smoke tests | N passed |
| Post-extraction smoke tests | N passed |
| User-visible behavior | Unchanged (verified by smoke tests) |

## Files Changed

| File | Change | RF case |
|------|--------|---------|
| src/... | new/edit/delete | RF-00X |

## Sub-commits

| Commit | Description |
|--------|-------------|
| abc1234 | W{n}.1 — <description> |
| def5678 | W{n}.2 — <description> |

## Traceability Matrix Update

- RF-00X: Status → Done, Commit → <hash>, Result → <outcome>
- RF-00Y: Status → Prepared, Commit → <hash>, Result → <outcome>
- Wave W{n}: Status → Done, Merge commit → <hash>

## Self-Correction (if any)

(If wave triggered self-correction per 3.3 Golden Rule, document here.
Otherwise: "None.")

## نتیجه (Result)

نتیجه: <X Violation حذف شد، Y RF بسته شد، Z مورد برای Wave بعد باقی ماند.>
```

## Rules

1. **Every wave must have a report** — no exceptions.
2. **Goal sentence is mandatory** — at the top, before any other content.
3. **Result sentence is mandatory** — at the bottom, after all evidence.
4. **Diff report is generated, not hand-written** — use `scripts/wave-diff.mjs`.
5. **Status vocabulary matches 3.6** — Planned / Prepared / In Progress / Done.
6. **Self-correction section is mandatory** — write "None." if nothing happened.
7. **Report is committed in the same merge commit as the wave** — not after.
