# 1.3 Refactor Strategy

## Priority Order (evidence-based)

### Priority 1: Break domain → state (16 violations)
**Why first:** This is the easiest direction to fix. Services should
not import store. This can be done with ports/adapters.

**Method:** Create port interfaces for the 16 service→store imports.
Services depend on ports. Adapters implement ports by delegating to
store. Wire in main.tsx.

**Risk:** Low — mechanical migration. Each service gets a port.

### Priority 2: Break domain → ui (6 violations)
**Why second:** Services should not know about UI components.

**Method:** Extract the 6 imports (mostly toast/useMessageEditor) into
ports or callback parameters.

**Risk:** Low — only 6 imports.

### Priority 3: Break state → ui (3 violations)
**Why third:** Store should not import components.

**Method:** The 3 imports are likely toast notifications from store
slices. Move to event/notification pattern.

**Risk:** Low — 3 imports.

### Priority 4: Break state → domain (28 violations)
**Why last:** This is the hardest. Store contains business logic that
needs to move to services. Moving logic changes behavior.

**Method:** Extract each of the 28 service imports from store into
service-layer functions. Store calls services via callbacks or events,
not direct imports.

**Risk:** HIGH — behavior change. Needs careful testing.

### Priority 5: Create persistence gateway
**Why:** 11 files access idb-keyval. No single point of control.

**Method:** Create a persistence service that wraps idb-keyval. All
other modules import from this service.

**Risk:** Medium — data format compatibility.

### Priority 6: Split God Files
**Why:** 13 files >500 lines. Maintainability risk.

**Method:** Extract sub-modules. Keep re-exports for backward compat.

**Risk:** Medium — cross-dependencies in God Files.

## Recommended Phase 2 Design Order

1. Port design for domain → state (Priority 1)
2. Port design for domain → ui (Priority 2)
3. Notification port for state → ui (Priority 3)
4. Service extraction plan for state → domain (Priority 4)
5. Persistence gateway design (Priority 5)
6. God File split plan (Priority 6)
