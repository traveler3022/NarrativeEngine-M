# 2.1 Refactor Target Validation

**Source:** Phase 0 artifacts (0.15, 0.16)
**Protocol:** Evidence-First

## Candidate Targets (from 0.16)

13 files >500 lines were identified as refactor candidates. This step
validates each one: is it ACTUALLY a problem, or just a large file
that's well-organized?

## Validation Criteria

A file is a VALID refactor target if it meets at least 2 of:
1. **Multiple responsibilities** — does more than one domain thing
2. **Multiple consumers** — imported by many unrelated modules
3. **High churn risk** — changes here risk breaking unrelated features
4. **Test isolation impossible** — can't be tested without full system
5. **Cross-boundary violations** — imports from wrong layers
